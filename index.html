Frequently Asked Questions

1. The difference between var, let, and const?
var is function-scoped, and can be redeclared and reassigned. let is block-scoped, can be reassigned, but not redeclared. const is block-scoped, and cannot be redeclared or reassigned. Use const by default, let for reassignment, and avoid var
2. The difference between map(), forEach(), and filter()
map() creates a new array by applying a function to each element. forEach() executes a provided function once for each array element, but doesn't create a new array. filter() creates a new array with elements that pass a test implemented by the provided function.
3. Explain arrow functions and how they are different from regular functions
Arrow functions offer a more concise syntax for writing function expressions. They differ from regular functions in their this binding, which inherits from the surrounding scope rather than being dynamically bound. Also, arrow functions cannot be used as constructors.  
4. how JavaScript Promises work?
JavaScript Promises represent the eventual completion (or failure) of an asynchronous operation and its resulting value for this. They have three states: pending, fulfilled, or rejected.Promises allow you to handle asynchronous operations in a cleaner, more readable way using .then() for success and .catch() for errors, avoiding callback hell.
5. How closures work in JavaScript?
Closures in JavaScript allow a function to access variables from its outer (enclosing) scope, even after the outer function has finished executing. This happens because the inner function retains a reference to the outer function's scope, creating a "closure" around those variables. This enables data encapsulation and the creation of private variables
